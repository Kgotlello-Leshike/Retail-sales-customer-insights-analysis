# 📌 Project Overview

This project analyzes a **Retail Sales & Customer Demographics dataset** to uncover insights into customer behavior, product performance, and seasonal sales trends.

The analysis combines **SQL queries, Python data analysis, and visualizations** to transform raw transactional data into actionable business insights.

Processed datasets from this analysis can also be used to build **interactive dashboards in Power BI**.

# 🎯 Project Objectives

Using **SQL and Python**, the project answers several business questions:

1. 💰 Total revenue and units sold per product category  
2. 🗓️ Monthly and quarterly sales trends  
3. 👑 Top customers by total spending and frequent transactions  
4. 👥 Customer purchasing behavior by demographics (age and gender)  
5. 📈 Identification of high-value transactions  
6. 🏷️ Product category performance across time periods  

These insights simulate **real-world retail analytics tasks** such as demand forecasting, marketing strategy, and customer segmentation.

# 📊 Key Insights

The analysis revealed several interesting patterns:

- **Electronics and Clothing** consistently generate the highest revenue.
- Sales show **seasonal spikes in May, October, and December**.
- A small group of **high-value orders contributes disproportionately to revenue**.
- Customers aged **26–55 generate the majority of revenue**.
- Average basket size is **approximately 2–3 items per order**.

# 🚀 Quick Start

To run this project yourself:

1. Open the notebook in **Google Colab** or **Jupyter Notebook**
2. Run all cells from top to bottom
3. The dataset will automatically download using **KaggleHub**
4. SQL queries will execute inside an **SQLite in-memory database**
5. Visualizations and insights will be generated throughout the notebook

Optional:
The exported datasets can be imported into **Power BI** for dashboard creation.

# ⚠️ Data Limitations

Since this dataset is synthetic, several limitations exist:

- The dataset contains **1000 transactions**, limiting long-term trend analysis.
- Some transactions show **identical values**, which may indicate synthetic or capped data.
- The dataset ends in **January 2024**, meaning later quarterly analysis is incomplete.
